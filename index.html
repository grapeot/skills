Core Philosophy

Platform-Agnosticism

Unlike vendor-locked extensions in specific IDEs, our skills are plain Markdown instructions with optional CLI scripts. They run seamlessly in Cursor, Claude Code, OpenCode, or any custom terminal agent.

Process & Result Certainty

Instead of relying on fuzzy prompts, our skill configurations define strict execution steps, error-handling heuristics, and checklists to guarantee highly predictable agent outcomes.

Files-as-Interfaces

All inputs and outputs are recorded as plain text files in your local workspace. This eliminates complex server infrastructure, keeping agent changes stateful and reviewable via Git.

One-Click Agent Installation Protocol

Click copy, paste this instructions prompt block directly into your agent terminal or workspace rules to guide your agent in setting up any repository-based skill below.

Install this public skill repo into my workspace: [GitHub Repository URL] Start from my workspace AGENTS.md. Follow any WORKSPACE.md routing rules. Clone or vendor the repo under an appropriate project directory. Expose exactly one root skill to my global rules/skills/ directory via a relative symlink. Keep private aliases, local paths, and credentials in a local overlay, not in the public repo.
