核心理念

01 / 跨平台

平台无关性

与特定 IDE 中厂商绑定的插件不同，我们的技能是纯 Markdown 指令加可选的 CLI 脚本。它们可以在 Cursor、Claude Code、OpenCode 或任何自定义终端 Agent 中无缝运行。

02 / 确定性

过程与结果确定性

我们不依赖模糊的 Prompt，而是通过技能配置定义严格的执行步骤、错误处理启发式规则和核对清单，从而保证 Agent 输出的高度可预测性。

03 / 文件

文件即接口

所有的输入 and 输出都作为纯文本文件记录在本地工作区中。这消除了复杂的服务器基础设施，保持了 Agent 变更的有状态性，并可通过 Git 进行审查。

常见问题

复制该技能的 GitHub URL 并在终端告诉你的 Agent：“阅读此仓库的规则，将其克隆至 adhoc_jobs/ 下，在 rules/skills/ 中映射一个相对软链接，并在 rules 索引中注册。” Agent 会自动拉取代码并进行链接。
公开仓库仅保留通用的代码契约。你所有的本地路径、API 凭证和个人别名都存储在工作区的私有覆盖（overlay）文件中，确保没有隐私数据泄建。
不需要。技能要么是纯文本指南，要么是简单的 Python CLI 封装。它们在 Agent 可以派生的任何终端会话中原生执行。

单行安装

无需特定 IDE 的斜杠命令，无需为了安装技能而调用另一个技能。点击任意条目上的 复制 并粘贴给你的 Agent —— 复制的 Prompt 中已包含了该技能类型所需的完整安装流程。
