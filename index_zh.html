平台无关的模块化规则与终端 CLI 工具目录，遵循「渐进式披露」哲学，让 AI Agent 完美对齐人类判断。

核心哲学

平台无关（Platform-Agnostic）

不同于绑定特定 IDE 的插件，我们的技能是纯 Markdown 指南与轻量 CLI 脚本。这使得它可以在 Cursor、Claude Code、OpenCode 或任何自定义终端 Agent 中无缝运行。

过程与结果的确定性

不依赖模糊的 Prompt 聊天，我们的技能文件定义了严密的执行步骤、异常处理分支与交付核对清单，确保 Agent 执行特定任务时结果高度 predictable。

文件即接口 (Files-as-Interfaces)

技能与工作区的交互通过读写文件进行。输入和输出均为工作区内的普通文本文件，天然适配 Git 版本控制，可随时进行审计、比对和版本回滚。

常见问题解答 (FAQ)

复制技能的 GitHub URL，并在终端 Agent 对话中输入：“读取此仓库的规则，克隆到 adhoc_jobs/，在 rules/skills/ 建立相对软链接，并注册进索引文件。” Agent 便会自动完成全部挂载。

公共仓库中只定义通用的代码和接口契约。而你本地的私有路径、API 密钥以及联系人别名等敏感数据，均保存在本地私有的 overlay 配置文件中，避免泄漏风险。

不需要。我们的技能要么是纯文本的操作规则，要么是免编译的 Python CLI 脚本。Agent 在终端会话中可以直接以原生命令行工具的形式来执行它们。

一键 Prompt 安装协议

点击复制此 Prompt 块并直接粘贴至你的 Agent 对话框，引导 Agent 在工作区中自动下载并配置下方列出的任何工具型技能。

Install this public skill repo into my workspace: [GitHub Repository URL] Start from my workspace AGENTS.md. Follow any WORKSPACE.md routing rules. Clone or vendor the repo under an appropriate project directory. Expose exactly one root skill to my global rules/skills/ directory via a relative symlink. Keep private aliases, local paths, and credentials in a local overlay, not in the public repo.
