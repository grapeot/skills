查看源仓库 ↗

核心理念

01 / 跨平台

平台无关性

与特定 IDE 绑定的插件不同，我们的技能是纯 Markdown 指令加可选的 CLI 脚本。它们可以在 Cursor、Claude Code、OpenCode 或任何自定义终端 Agent 中直接运行。

02 / 确定性

过程与结果可控

我们不依赖模糊的 Prompt，而是通过具体的技能定义清晰的执行步骤、容错逻辑和核对清单，让 Agent 的输出结果稳定可预测。

03 / 文件

文件即接口

所有的输入与输出都通过工作区内的纯文本文件进行交互。这不需要复杂的服务器支持，所有变更都留在本地，可随时通过 Git 审计与回滚。

常见问题

复制技能的 GitHub 链接并发送给 Agent：‘阅读这个仓库的规则，克隆到 adhoc_jobs/ 下，在 rules/skills/ 中建立相对软链接，并在 rules 索引里注册。’ Agent 就会自动完成代码下载与链接映射。
公开仓库只定义通用的技术契约与 CLI 接口。你所有的本地路径、API 凭证和个人别名都存储在本地工作区的私有覆盖（overlay）文件里，避免任何敏感数据外泄。
不需要。这些技能要么是纯 Markdown 操作指南，要么是轻量的 Python CLI 脚本。Agent 可以在终端会话中直接运行它们。

一键复制安装

不用记忆复杂的斜杠命令，也不需要安装器。只需点击任意条目上的 复制 按钮并将 Prompt 粘贴给 Agent —— 复制的内容已包含了此类型技能所需的全部安装指引。
